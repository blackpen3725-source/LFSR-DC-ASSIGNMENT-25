LFSR — Theory

Introduction

A Linear Feedback Shift Register (LFSR) is a sequential digital circuit that generates a sequence of binary numbers by shifting the contents of a register and feeding back a linear combination of its previous states.

The feedback function is linear and is typically implemented using the exclusive-OR (XOR) operation, producing a sequence that appears random but is completely deterministic for a given seed.

Structure and Operation

An LFSR consists of a chain of flip-flops forming a shift register; selected outputs called taps are combined through XOR gates and fed back to the register input.

On each clock pulse, all bits shift one position toward the output end and the feedback bit is inserted into the first flip-flop, creating a repeating pseudo-random sequence.

For an n‑stage LFSR, the maximum possible period of the output sequence is (2^n - 1) bits, and this maximum length occurs only when the taps correspond to a primitive polynomial over GF(2).

Types of LFSR

  • Fibonacci LFSR: Feedback from selected taps is XORed and applied only to the input of the first flip-flop; this is the most common textbook form.
  • Galois LFSR: Feedback is distributed internally to several flip-flops, allowing faster hardware implementations while generating the same sequences as the corresponding Fibonacci form.

Components Required

  • Flip-flops (D or T) to store each bit of the n‑bit register.
  • XOR gates to combine tap outputs and form the feedback bit over GF(2).
  • Clock generator providing periodic pulses that trigger each shift.
  • Seed or parallel-load switches to set a non-zero initial state for the register.
  • Output indicators such as LEDs or probes to observe the state and bitstream.
  • Power supply or digital simulator to implement and test the design.

Working Steps

  • Choose the register length n and appropriate tap positions for the required sequence.
  • Load a non-zero seed into the register using switches or parallel-load control.
  • On each clock edge, shift all bits toward the output stage.
  • Compute the feedback as the XOR of the tapped bits and insert it at the input flip-flop.
  • Read the output bit from the last flip-flop each cycle to build the output bitstream.

If the register state ever becomes all zeros, the LFSR will remain locked in that state, so the all-zero seed must be avoided.

Worked 4‑bit Example

Consider a 4‑bit LFSR with taps at Q3 and Q2 and seed 1001; the feedback bit is computed as Q3 ⊕ Q2.

  • Initial state 1001 → feedback = 1 ⊕ 0 = 1; after shifting, the new state is 1100 and the output bit is 1.
  • Next state 1100 → feedback = 1 ⊕ 1 = 0; after shifting, the state becomes 0110 with output 0.
  • Next state 0110 → feedback = 0 ⊕ 1 = 1; after shifting, the state is 1011 with output 0.

Continuing this process eventually produces a repeating deterministic bitstream whose period depends on the number of stages and chosen taps.

Applications

  • Pseudo-random number generation for simulations and spread-spectrum systems.
  • Spreading codes in CDMA and other spread-spectrum communication schemes.
  • Error detection via CRC hardware, which uses LFSR-like polynomial division.
  • Built‑in self-test (BIST) where LFSRs create test patterns and compact responses.
  • Cryptographic stream ciphers that combine multiple LFSRs with nonlinear functions.
  • Data whitening and scramblers to remove long runs of identical bits on serial links.

Advantages

  • Very simple structure and easy implementation in hardware or software.
  • Requires minimal resources while generating long pseudo-random sequences at high speed.
  • Reliable for communication systems, testing, and error-detection circuits.

Disadvantages

  • Sequence is predictable when taps and seed are known, limiting security.
  • Randomness quality is lower than that of advanced cryptographic generators.
  • Improper initialization can cause the LFSR to lock into the all-zero state.
