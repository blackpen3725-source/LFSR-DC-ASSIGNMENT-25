VIVA-VOCE For LFSR

1. What is an LFSR?
Ans: A shift register that generates pseudo-random sequences using a linear feedback function, typically XOR.
2. How does an LFSR generate sequences?
Ans: By shifting bits through register stages and feeding back the XOR sum of specific bit positions (taps) into the input.
3. Difference between Fibonacci and Galois LFSRs?
Ans: Fibonacci uses an external XOR chain for feedback; Galois places XOR gates internally between shift stages.
4. How to determine the period of an LFSR?
Ans: The period is determined by the feedback polynomial; a primitive polynomial of degree $n$ results in a period of $2^n - 1$.
5. Role of the feedback polynomial?
Ans: It defines the mathematical relationship between bits; it dictates the sequence length and randomness properties.
6. How to design an LFSR for a given polynomial?
Ans: Map the polynomial coefficients to tap positions, connecting those stages to XOR gates in a feedback loop.
7. Significance of a primitive polynomial?
Ans: It ensures the LFSR cycles through all possible $2^n - 1$ non-zero states (Maximal Length Sequence).
8. LFSRs in cryptography?
Ans: Used as keystream generators in stream ciphers due to their high speed and simple hardware requirements.
9. LFSR vs. PRNG?
Ans: An LFSR is a hardware-efficient type of Pseudo-Random Number Generator (PRNG); other PRNGs can be non-linear or purely software-based.
10. How to test LFSR randomness?
Ans: Through statistical tests like the Golomb postulates, frequency tests, and autocorrelation analysis.
11. What are common applications?
Ans: Wireless communication (spreading), digital testing (BIST), checksums (CRC), and bit scrambling.
12. Relation to cyclic codes?
Ans: LFSRs implement the division of polynomials, which is the core logic behind generating and checking cyclic codes.
13. Impact of initial state?
Ans: Defines the starting point of the sequence. An all-zero initial state results in no output (stuck at zero).
14. Implementing LFSR in hardware?
Ans: Constructed using D-flip-flops for shift stages and XOR logic gates for feedback.
15. Max period of an n-bit LFSR?
Ans: $2^n - 1$ states.
16. How do taps affect behavior?
Ans: They change the feedback polynomial; the wrong taps result in a short, non-maximal cycle.
17. Internal vs. External XOR?
Ans: Internal (Galois) is faster because XOR gates are distributed; External (Fibonacci) has more gate delay in the feedback path.
18. Use in error detection?
Ans: They perform the polynomial division required to calculate the CRC remainder.
19. Can LFSRs encrypt?
Ans: Yes; by XORing the generated sequence (keystream) with the plaintext data bits.
20. Role in BIST?
Ans: Generates exhaustive test patterns to find faults in logic circuits without external equipment.
21. How to analyze sequences?
Ans: Using state transition diagrams, shift-and-add properties, and Z-transforms.
22. What is a characteristic polynomial?
Ans: A polynomial representation $f(x)$ that describes which register stages contribute to the feedback.
23. Converting configurations?
Ans: By mirroring the tap positions and rearranging XOR placement according to the reciprocal polynomial.
24. Limitations in crypto?
Ans: Their linearity allows attackers to predict future bits using the Berlekamp-Massey algorithm if a few bits are known.
25. Generating m-sequences?
Ans: Requires an $n$-bit LFSR loaded with a non-zero seed and a primitive polynomial.
26. Connection to shift registers?
Ans: An LFSR is a standard shift register enhanced with a logic-based feedback loop.
27. How to determine if maximal?
Ans: By verifying the period equals $2^n - 1$ through simulation or polynomial testing.
28. Impact of zero initial state?
Ans: The register stays at all-zero state because XOR(0,0)=0.
29. Use in digital signatures?
Ans: Indirectly, as part of PRNGs used for key generation or salt.
30. Combining LFSRs?
Ans: Yes, techniques like Geffe or Stop-and-Go generators combine LFSRs to increase non-linearity and period.
31. Role in scrambling?
Ans: Breaks up long sequences of 1s/0s to ensure clock synchronization at the receiver.
32. Contribution to stream ciphers?
Ans: Acts as the source of the pseudo-random bitstream.
33. LFSR vs. NLFSR?
Ans: LFSR uses linear gates (XOR); NLFSR uses non-linear gates (AND/OR), making them harder to analyze/attack.
34. Implementing in software?
Ans: Using bitwise operators: `out = (state >> n) & 1; state = (state << 1) ^ feedback`.
35. Common hardware faults?
Ans: Stuck-at-0 or Stuck-at-1 faults in the flip-flops or gates that break the cycle.
36. Handling overflow?
Ans: LFSRs do not overflow; they naturally wrap around to their initial state after one full period.
37. Significance in comms?
Ans: Essential for CDMA (multiplexing) and GPS signal identification.
38. Choosing taps?
Ans: Selected from look-up tables of primitive polynomials for the required register length.
39. Relation to PN sequences?
Ans: LFSRs are the primary hardware method for generating Pseudo-Noise (PN) sequences.
40. Randomness testing?
Ans: Verified using NIST statistical suites which check for bias and patterns.
41. Parallelization in hardware?
Ans: Achieved by unwinding the feedback logic to compute $k$ bits per clock cycle instead of one.
42. Impact of length on randomness?
Ans: Longer registers provide higher complexity and longer periods, making the sequence appear more "random" over time.
43. Verifying correctness?
Ans: By comparing the simulation output against a known-good sequence for a specific polynomial and seed.
44. Spread spectrum role?
Ans: It spreads a narrow-band signal over a wider bandwidth using a high-speed PN sequence.
45. Calculating output for input?
Ans: Apply the XOR feedback to the current state bits to find the bit shifted into stage 1.
46. LFSR vs. CA?
Ans: LFSR has global feedback; Cellular Automata (CA) use local rules based on neighboring cells.
47. BIST for memory?
Ans: Used as address generators to cycle through all memory locations during a test.
48. Designing for specific length?
Ans: Use an $n$-stage register where $2^n-1 \geq$ desired length, then truncate or adjust feedback.
49. Common applications?
Ans: Ethernet CRC, Bluetooth scrambling, and Wi-Fi security protocols.
50. Speed comparison?
Ans: LFSRs are extremely fast in hardware (Ghz range) but slower in software than word-oriented PRNGs.
